## 修复 MusicPlayerScreen 歌词显示问题

文件：`lib/features/player/music_player_screen.dart`

### 问题
用户反馈：歌词界面播放音乐时歌词一直闪烁刷新，并且只在顶部和底部显示一行歌词，中间一大块是空的。

### 根因
`_LyricsView` 里的 `ShaderMask` 使用了错误的 `blendMode`：
- 原来用的是 `BlendMode.dstOut`
- 配合 `[transparent, black, black, transparent]` 渐变
- `dstOut` 的语义是"擦除 source alpha 高的位置"——结果是中间黑色区域被擦除、顶部底部透明区域保留
- 表现为"顶部底部各一行、中间一大块空"

### 修复
1. `BlendMode.dstOut` → `BlendMode.dstIn`（保留黑色区域，擦除透明区域）
2. `stops` 从 `[0.0, 0.08, 0.92, 1.0]` 扩到 `[0.0, 0.20, 0.85, 1.0]` —— Apple Music 风格的边缘渐隐（顶部 20% + 底部 15%）
3. `Ticker (60fps)` → `Timer.periodic(50ms)` —— 减少重建频率，避免闪烁
4. `AnimatedOpacity` → `AnimatedDefaultTextStyle` —— 字号/字重/颜色同时过渡，更顺滑
5. 增加 Apple Music 风格的距离衰减透明度：active=1.0, distance=1→0.55, =2→0.38, 其他→0.22
6. `_activeLine` 距离算法：把 active 行滚到视口中央（之前公式有正负号错误）
7. `ListView` padding 上下加大到 100px，确保 active 行能居中
8. 加 `RepaintBoundary` 隔离重绘

### Apple Music 风格设计要点
- 当前行：22pt 粗体白色
- 距离 1：16pt 普通 55% 白色
- 距离 2：16pt 普通 38% 白色
- 更远：16pt 普通 22% 白色
- 顶部底部 15-20% 渐变到背景
- 切换过渡：400ms easeOutCubic

### 3. 移动端长按改为弹菜单 + 加"多选"入口
**`lib/widgets/media_context_sheet.dart`** —— 加 "多选" `ListTile`（带"check_box_outlined" 图标 + "进入多选模式，可批量操作" 副标题），需传入 `selectionId` 参数。

**`lib/features/media/media_type_screen.dart`** + **`lib/features/folders/folder_detail_screen.dart`** —— 长按逻辑改为：`!kIsWeb && !Platform.isWindows` 时弹 `MediaContextSheet`（含"多选"入口），否则保持原行为。`MediaContextSheet.show` 传 `selectionId`。

### 4. 收藏功能 bug 修复
**`lib/widgets/batch_action_bar.dart`** —— `_favorite()` 改写：写完 `db.setFavorite(path, true)` 后调用 `ref.read(mediaProvider.notifier).reloadFromDatabase()` 让 in-memory state 同步刷新，避免"DB 已改但 UI 还显示未收藏"。删除按钮也加上 `reloadFromDatabase()`。

### 5. 切换排列方式灰屏 bug 修复
**`lib/widgets/async_view.dart`** —— `value.when()` 加 `skipLoadingOnRefresh: true` 和 `skipLoadingOnReload: true`，关键修复：**只在首次加载时显示 loading spinner，refresh/reload 时保留旧 data**。这正是切换 view 时屏幕灰屏的根因 —— `searchedMediaProvider` 因为 watch 了整个 `settingsProvider`，切 view 时会重跑，瞬间变成 `AsyncLoading`，原 `AsyncView` 直接渲染 `CircularProgressIndicator`（占满视口的灰屏），现在保留旧 data。

## 修复 Android 删除 + Windows 灰屏诊断增强

### Android 长按菜单删除失败修复
**`lib/services/trash_manager.dart`** —— fallback 分支 `file.rename(dest)` 在 Android 11+ scoped storage 跨存储卷会失败。改为 try-catch rename，失败回退到 `copy + delete`。

### Windows 灰屏诊断增强（不是修复，是暴露错误）
1. **`lib/main.dart`** —— 用 `runZonedGuarded` 包整个 main，捕获所有未处理异常（同步/异步/microtask），打印到 stderr
2. **`lib/main.dart`** —— 加 `FlutterError.onError`，把 framework 错误（build/layout/paint）也打印到 stderr
3. **`lib/main.dart`** —— `ErrorWidget.builder` 显示红色屏 + exception + stack trace（替代 release 模式默认的灰色屏）
4. **`lib/widgets/media_thumbnail.dart`** —— 把 `_VideoThumbnail._loadFromDiskCache` 和 `_generateThumbnail` 里的空 `catch (_) {}` 改为 `catch (e, st) { debugPrint(...) }`，暴露原本被吞掉的异常
5. **`lib/widgets/media_thumbnail.dart`** —— 路径拼接从字符串插值 `'${base.path}/lumiluna_thumbs'` 改为 `p.join(base.path, 'lumiluna_thumbs')`（package:path），消除 Windows 上 `\` 和 `/` 混用
6. **`lib/features/home/home_screen.dart`** —— 3 处 `ref.watch(settingsProvider)` 改为 `select`，避免切排序时整个 HomeScreen 重建

- 如果看到**红色错误屏**：截图发回来，错误信息 + stack trace 直接定位根因
- 如果还是**灰色屏**：说明是 paint phase 异常或 native 资源问题，需要进一步排查（可能是 media_kit 的 native 资源冲突）
