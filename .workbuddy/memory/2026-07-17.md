## 修复 MusicPlayerScreen 歌词显示问题

文件：`lib/features/player/music_player_screen.dart`

### 问题
用户反馈：歌词界面播放音乐时歌词一直闪烁刷新，并且只在顶部和底部显示一行歌词，中间一大块是空的。

### 根因
`_LyricsView` 里的 `ShaderMask` 使用了错误的 `blendMode`：
- 原来用的是 `BlendMode.dstOut`
- 配合 `[transparent, black, black, transparent]` 渐变
- `dstOut` 的语义是"擦除 source alpha 高的位置"——结果是中间黑色区域被擦除、顶部底部透明区域保留
- 表现为"顶部底部各一行、中间一大块空"

### 修复
1. `BlendMode.dstOut` → `BlendMode.dstIn`（保留黑色区域，擦除透明区域）
2. `stops` 从 `[0.0, 0.08, 0.92, 1.0]` 扩到 `[0.0, 0.20, 0.85, 1.0]` —— Apple Music 风格的边缘渐隐（顶部 20% + 底部 15%）
3. `Ticker (60fps)` → `Timer.periodic(50ms)` —— 减少重建频率，避免闪烁
4. `AnimatedOpacity` → `AnimatedDefaultTextStyle` —— 字号/字重/颜色同时过渡，更顺滑
5. 增加 Apple Music 风格的距离衰减透明度：active=1.0, distance=1→0.55, =2→0.38, 其他→0.22
6. `_activeLine` 距离算法：把 active 行滚到视口中央（之前公式有正负号错误）
7. `ListView` padding 上下加大到 100px，确保 active 行能居中
8. 加 `RepaintBoundary` 隔离重绘

### Apple Music 风格设计要点
- 当前行：22pt 粗体白色
- 距离 1：16pt 普通 55% 白色
- 距离 2：16pt 普通 38% 白色
- 更远：16pt 普通 22% 白色
- 顶部底部 15-20% 渐变到背景
- 切换过渡：400ms easeOutCubic

- `flutter analyze lib/features/player/music_player_screen.dart` 通过，无 issue

## 用 flutter_lyric 重构歌词 + MD 控件 + 双语翻译 + shuffle

### 新增依赖
- `flutter_lyric: ^3.0.7` —— Apple Music 风格歌词组件，只依赖 plugin_platform_interface，零音频冲突

### 改动文件

**`lib/models/lyrics.dart`** —— 重写解析逻辑，支持多时间标签 LRC（`[01:23.45][02:34.56]text`），二分查找 active line

**`lib/services/lyrics_service.dart`** —— 新增 `loadRawLyrics()` 返回原始 LRC 字符串（flutter_lyric 需要原始文本自己解析）

**`lib/services/lyrics_translation_service.dart`** (新建) —— 按优先级查找翻译文件：`.zh.lrc` > `.translation.lrc` > `.zh-CN.lrc` > `.zh-Hans.lrc` > `.translate.lrc`

**`lib/providers/lyrics_provider.dart`** —— 重写：
- `lyricsProvider` 改为返回 `FutureProvider<String?>`（原始 LRC 文本）
- 新增 `lyricsTranslationProvider` 返回翻译文本
- 都依赖 `currentMediaProvider`（只随曲目变化，不随 position tick）

**`lib/providers/player_provider.dart`** —— 加 shuffle 支持：
- `PlaybackState` 加 `shuffling` 字段
- `PlaybackController` 加 `toggleShuffle()`、`_randomIndexExcluding()`
- 重写 `next()`/`previous()`：shuffle 开时跳随机 index
- 监听 `player.stream.completed`：shuffle 开时自动跳随机
- `toggleLoop()` 在 shuffle 开时保持 PlaylistMode.none（让 completed 触发我们的 shuffle 逻辑）
- 新增 `currentMediaProvider` —— 只随曲目 path 变化的轻量 provider

**`lib/features/player/music_player_screen.dart`** —— 大改：
- `_LyricsView` 完全重写：用 `LyricController` + `LyricView`，`player.stream.position.listen(controller.setProgress)` 推模式驱动
- `_LyricsPanel` (新增)：歌词加载面板，包含翻译切换按钮
- `_TranslationToggle` (新增)："仅原文"/"原文+翻译" 切换 chip
- `_NarrowLayout` 改为 StatefulWidget：点击封面切换歌词覆盖层，移动端适配
- `_PlaybackRegion`：直接 watch providers，不再接收 lyricsAsync 参数；加播放列表 onTap 修复
- `_LyricsOrQueue`：同上
- `_Controls`：加 shuffle 按钮（MD 风格）、play/pause 用 `Material` + `InkWell`（ripple effect）、倍速按钮从 AppBar 移入控件行
- `_SpeedButton` (新增)：倍速选择器
- `_Playlist`：修复 onTap bug（原来 onTap 回调是空的）
- 移除 AppBar 的倍速 PopupMenuButton

### l10n
- `app_en.arb` / `app_zh.arb`：加 `shuffleTooltip`

### 验证

## 移动端布局：PageView 滑动 + 去掉底部歌词面板

### 问题
用户截图显示移动端底部仍残留"歌词/播放列表"切换按钮和歌词显示区域，与"点击封面看歌词"的设计冲突。

### 改动
**`lib/features/player/music_player_screen.dart`**：
- `_NarrowLayout` 重写为 `PageView`（2 页）：
  - Page 0 = 播放器主界面（封面 + info + seek bar + controls），点击封面显示歌词覆盖层

- `_Playlist` 的 onTap 已在前一轮修复，现在通过 `_PlaylistPage` 正确调用 `controller.jump(i)`

### 交互逻辑
