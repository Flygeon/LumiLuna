## Flutter 媒体库应用 (material_library) 实现

从绿地项目实现了完整可编译的 Flutter + Material 3 媒体库应用（Windows 桌面）。

- 技术栈：Flutter 3.44.6 stable / Dart 3.12.2；Riverpod 状态管理；media_kit 统一音视频播放（原生支持 Windows）；dart:io + file_picker 本地扫描；shared_preferences 持久化。
- 关键决策：photo_manager 不支持 Windows，故改用文件系统递归扫描（默认 Pictures/Videos/Music + 自定义文件夹）。
- 功能：媒体浏览（网格/列表切换、isolate 扫描）、集合播放（图片查看器/视频连播/音乐播放列表）、媒体管理（相册/文件夹/日期分组 + 搜索）、MaterialApp+NavigationBar+深浅主题。
- 27 个 lib/ 源文件 + pubspec.yaml + analysis_options.yaml + .gitignore + README.md + .github/workflows/build-windows.yml。
- CI 在运行时用 `flutter create --platforms=windows` 生成平台层后 `flutter build windows --release`（仓库存源码不存 windows/ 脚手架）。
- 状态：代码已写完，正用本机 flutter pub get + flutter analyze 做真实校验。

## 三个 Bug 修复 + 字体依赖冲突解决
- Bug1 视频缩略图不显示：原 MediaThumbnail 对视频直接返回图标占位。改用 fc_native_video_thumbnail（Windows 原生支持）新增 _VideoThumbnail 子组件，按需抽帧并按路径哈希缓存到临时目录（lumiluna_thumbs/），失败回退图标占位。文件：lib/widgets/media_thumbnail.dart。
- Bug2 返回主界面视频未暂停：VideoPlayerScreen 用 PopScope(onPopInvokedWithResult) 在页面 pop 时调用 controller.pause()。文件：lib/features/player/video_player_screen.dart。
- Bug3 字体问题：引入 luoyi_flutter_font 失败——pub 解析冲突：file_picker(>=8.1.0) 需 web^1.0.0，而 luoyi_flutter_font(所有版本,已 discontinued) 需 web^0.5.1，永远冲突。改用零依赖方案：把用户提供的 SourceHanSansSC-Regular-2.otf（16MB，思源黑体=Noto Sans SC 同源）放到 assets/fonts/，pubspec 声明 family 'NotoSansSC'，AppTheme.fontFamily='NotoSansSC'，main.dart 移除 FlutterFont.initFont。
- 依赖：pubspec 新增 fc_native_video_thumbnail ^1.0.0（已解析成功，Changed 1 dependency）。落库注意：assets/fonts/*.otf 必须 git add 提交，否则 CI 找不到字体资源会失败。
- 校验状态（本机）：flutter pub get 已成功；flutter analyze 验证进行中/待确认。

## 选项卡切换动画
- home_screen.dart：把 body 的 IndexedStack 换成 PageView + PageController。导航栏 onDestinationSelected 改为 _onTabSelected -> setState + _pageController.animateToPage(index, 320ms, Curves.easeInOutCubic)。physics: NeverScrollableScrollPhysics（仅点击驱动，避免与媒体列表纵向滚动冲突；保留滑动手势可作为后续可选增强）。PageView 保留已访问页面，滚动位置不丢。

## 选项卡动画掉帧（诊断，未修复）
- 现象：切换 tab 时动画明显掉帧。根因非曲线/时长，而是 PageView.animateToPage 切换机制：
  1) 滑动期间「源页 + 目标页」同时存活并渲染（home_screen.dart:124-135 固定 4 children），双画廊同帧合成，绘制开销约 2 倍。
  2) 首次进入某 tab 时目标页从零构建：图片 Image.file 批量解码 + 视频 tab 的 _VideoThumbnail.initState 同步发起 N 个原生 getVideoThumbnail 调用（media_thumbnail.dart:85-121），cacheExtent:600（media_grid_view.dart:41）放大首屏一次性触发的格子数，工作洪峰正好压在动画起点。
  3) 每页未包 RepaintBoundary，整个 body 每帧因 transform 变化而重绘。
- 待修复方向（用户尚未要求实现）：①每页 RepaintBoundary + 降 cacheExtent；②视频抽帧延后到该页成为激活页且动画结束后再做，先只解首屏可见格；③或改用 AnimatedSwitcher 淡入（同时只活一页，但丢滚动位置保持）。
- 非根因：Curves.easeInOutCubic / 320ms 时长（改时长无效）。
